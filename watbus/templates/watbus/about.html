The Story of watbus

watbus was created at the CUTC hackathon in May of 2013. It was originally coded in PHP with no webframework whatsoever. The developers thought it would be a good idea to continue with the project, but with a MVC architecture and a better language the next time around.

Marc chose to use the Django MVC web framework and SQLite (now MySQL) to store the enormous amount of transit data. He is responsible for manipulating the data and ensuring that bus scheduls are always complete and accurate. Marc also wrangled with Dreamhost and got Django and Python to co-operate on the shared server (Try it, it's painful.)

Holly used Javascript for the Google Maps API and various dynamic elements of the site. Bus stop data from the database was retrieved using JSON/AJAX and plotted on the map. Designing and styling watbus was done by Holly as well.

The current design is called "watbus Terminal". The developers coded almost the entirety of this website in Vim in the Terminal. Bus terminal... computer terminal... ha ha. How punny.
